## Infineon의 AURIX™ 보드로 MCU 프로그래밍 익히기

- 개발 보드: ShieldBuuddy TC275
- 개발 환경: AURIX Development Studio

`tricore` 보드로 3개의 코어가 있지만 CPU0의 한 개만 사용하여 실습

---
### 260219_INTERRUPT

🎯 과제

S3 버튼 입력 인터럽트를 활용하여 LED 깜빡임을 On/Off

⚙️ 특징
- `iLLD` 드라이버를 사용하지 않고 **레지스터 직접 제어** 방식
- `상태 기반(FSM)`으로 LED 동작 제어
  - STATE_BLINKING(깜빡임)
  - STATE_IDLE(정지) 
- `ERU(External Request Unit)`를 이용한 외부 인터럽트 설정
- `Timer` 대신 for문 기반 **Busy-wait** 방식으로 LED 깜빡임 주기를 제어
---
### 260223_TIMER

🎯 과제 목표

0.5초에 한 번씩 상태(state)가 변하는 신호등 만들기
- STATE_RED = 빨간불 ON
- STATE_BLUE = 파란불 ON
- STATE_BLUE_BLINK = 파란불점멸신호 (0.05초에한번씩깜빡임(toggle))

⚙️ 특징
- GPIO핀 초기화에서 `iLLD` 드라이버를 사용
- but ERU 초기화에서는 레지스터 직접 제어 방식 사용
- 정확한 타이밍을 위해 하드웨어 `클럭`을 이용

---
### 260223_FND
🎯 과제 목표

SW1/SW2 버튼 입력을 활용하여 4자리 FND(7-segment) 카운터 동작 제어
  - SW1: 카운터 동작 시작/정지
  - SW2: 카운터 초기화

⚙️ 특징

- `iLLD` 드라이버 사용으로 코드 안전성 ↑
- 두 버튼 입력에 대한 인터럽트 구현
- 상태 변수(g_state) 사용으로 FND 카운터 제어
- **7-Segment**는 `Common Anode` 기준

---

### 260224_SCHEDULE

🎯 과제 목표

- 앞서 만든 TIMER을 이용하여 `non-preemptive scheduler` 구현
- 4FND의 맨 아랫자리 부터 10ms/100ms/1s/10s 를 세대록 타이머를 사용

⚙️ 특징
- 4FND와의 `SPI` 통신에는 5개의 GPIO핀 사용(GND/VDD/SCLK/RCLK/DIO)

---

### 260224_ADC

🎯 과제 목표

- 가변저항의 ADC 결과에 따라 달라지는 값을 FND 회로에 출력
- 가변 저항 ADC 결과에 따라 (0, 1300, 2600, 4095)의 구간을 나누어 Red/Green/Blue 세 개의 LED를 켜서 구분
- 조도 센서를 가렸을 때 회로 내장 붉은 LED가 점등되고 가리지 않으면 내장 파란 LED가 점등

---

### 260225_MINI_PROJECT

🎯 과제 목표

**[REQ-01]**
- 시스템은 일반모드에서 가변저항의 ADC 입력값에 비례하여 0 ~200km/h 범위의 속도를 유지해야 한다.
- 시스템은 크루즈 모드 활성화 시 외부 입력과 무관하게 차량의 속도를 70km/h로 고정 유지
- 시스템은 비상 상태 진입 시 즉각적으로 차량의 속도를 0km/h로 감속 및 고정

**[REQ-02]**
- 시스템은 조도 센서를 통해 주변 밝기의 측정값이 임계치 미만일 경우 **LED D13**을 자동 점등
- 시스템은 차량 속도가 120km/h를 초과할 경우 **LED D12**을 점멸하고 **Buzzer**를 울려야 됨

**[REQ-03]**
- 시스템은 4-FND를 통해서 현재 주행 속도를 실시간으로 표기
- RBG LED를 사용하여 현재 시스템을 아래와 같이 표시
    - Normal: Green
    - Cruise: Blue
    - Emergency: Red

**[REQ-04]**
- 시스템은 스위치 1(P02.0) 입력을 감지하면 일반 모드와 크루즈 모드 간의 상태 전환
- 시스템은 스위치 2(P02.1) 입력을 감지하면 즉시 비상 상태로 전이하며 재입력시 일바 모드로 전환
- 시스템은 모든 스위치 입력에 대해서 즉각적인 반응성을 보장

**[REQ-05]**
- 시스템은 `비선점 스케줄러`를 기반으로 동작하며 각 태스크는 정의된 주기(1ms,10ms,100ms,1000ms)를 준수
